Apa itu Physical Model?

Tuesday, 21 September 2021

Pada artikel sebelumnya, kita sudah mempelajari apa itu pemodelan data dan Conceptual Data Model. Pada artikel ini, saya akan menjelaskan modelan data fisik atau sering disebut dengan Physical Data Model.

Apa itu Physical Data Model? Physical Data Model menggambarkan bagaimana caranya sistem basis data akan diimplementasikan menggunakan sistem DBMS tertentu. Model ini biasanya dibuat oleh pengembang [1]. Physical Data Model adalah model khusus database yang mewakili objek data relasional (misalnya, tabel, kolom, kunci utama dan kunci asing) dan hubungannya. Physical Data Model dapat digunakan untuk menghasilkan pernyataan DDL (Data definition language) yang kemudian dapat digunakan ke server database [2]. Dengan kata lain, physical data model menggambarkan bagaimana sebuah model akan dibuat dalam basis data.

Physical Data Model mewakili bagaimana data harus terstruktur dan terkait dalam DBMS tertentu sehingga penting untuk mempertimbangkan konvensi dan batasan DBMS yang Anda gunakan saat Anda mendesain ERD fisik. Ini berarti bahwa penggunaan tipe data yang akurat diperlukan untuk kolom entitas dan penggunaan kata khusus (reserved words) harus dihindari dalam penamaan entitas dan kolom [3].
